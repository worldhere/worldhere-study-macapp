# 机器管理 — 输入增加

## Context

机器管理当前有"新增"（单台）和"批量添加"（编号范围）两种方式。需要增加第三种"输入增加"，支持粘贴含机型+名称的文本，自动解析出机器名和任务类型。

## 入口

在机器管理面板"新增"和"批量添加"按钮旁边，加一个"输入增加"按钮。

## 弹窗

点击后弹出对话框，包含：
- textarea 文本框（供粘贴内容，如示例机器号.txt）
- 预览区（实时显示解析结果：机器名 → 机型 → 任务类型）
- "确认添加"按钮

## 解析规则（逐行处理）

1. 去除首尾空白，空行跳过
2. 检测前缀机型：行首匹配 machine_types 中的机型（BR1、BR2、Mini 等），识别为机型；无匹配则跳过该行
3. 检测括号后缀：匹配末尾的中/英文括号 `(...)` 或 `（...）`
4. 任务类型匹配：括号内容匹配已配置的 task_kinds → 机器名=去掉括号部分，任务类型=匹配值
5. 不匹配：整个行内容作为机器名，任务类型用默认值

## 后端

扩展 `/add_machines_batch`，支持每台机器携带独立 task_kind：
- 当前接口：`{type, task_kind, names: [...]}` — 所有机器共用 task_kind
- 改为：`names` 数组中每项可以是字符串（兼容旧行为）或 `{name, task_kind}` 对象

## 示例

| 输入 | 机型 | 机器名 | 任务类型 |
|------|------|--------|----------|
| BR2-10 | BR2 | BR2-10 | 默认 |
| BR2-11(DAgger) | BR2 | BR2-11 | DAgger |
| BR2-15（ASR） | BR2 | BR2-15 | ASR |
| BR2-20(未知) | BR2 | BR2-20(未知) | 默认 |

## Verification
- 粘贴示例机器号.txt 内容，确认预览显示正确解析
- 确认添加后机器列表出现新机器，任务类型正确
- 中英文括号均能识别
- 括号内容不匹配 task_kinds 时，完整名称保留
