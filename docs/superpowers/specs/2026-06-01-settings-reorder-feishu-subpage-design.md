1. 重排设置子页面顺序：班次设置提到最前，其他顺延
2. 从系统设置中拆分飞书相关内容，新开「飞书同步」管理子页面

## 飞书同步子页面布局

从上到下：

### 1. 飞书数据同步开关（顶部）

- **iOS 风格滑动开关**（不是勾选框），替代原来的 `<input type="checkbox" id="s-feishu-sync">`
- 默认关闭状态（灰色滑块靠左）
- 开启后滑块滑到右侧，背景变色（绿色）
- 右侧显示状态文字：「飞书同步已关闭」/「飞书同步已开启」
- 描述文字在开关左侧：「开启后将排班数据同步到飞书多维表格，关闭不会丢失已同步数据」

### 2. 状态指示器（开关下方）

- 关闭状态：黄色背景 → 🟡 未连接 · 尚未初始化
- 已连接状态：绿色背景 → 🟢 已连接 · Base: xxx... · X/Y 台 · 上次推送: X/Y (100%)
- 连接失败：红色背景 → 🔴 连接失败
- 包含距下次同步倒计时

### 3. 操作按钮

- **初始化同步**（蓝色主按钮）— 未初始化时显示
- **刷新状态** — 刷新连接和同步状态
- **立刻推送** — 点击后暂停自动推送 → 等待推送完成 → 重新计时
- **一键清理飞书表**（红色警告按钮，小字）— 开发用

### 4. 自动推送间隔设置

- Number input + "秒" 单位
- 默认值 30
- **disabled + 只读**，黄色「暂不可用」标签
- 描述文字：「功能预留，当前使用默认间隔 30 秒」
- 后端已有 `sync_interval_sec` 字段，前端 input 只读展示，将来放开 disabled 即可用

### 5. 异常标记选项（页面底部）

- 灰色虚线边框，整体降透明度
- 「暂不可用」标签
- 当前已配置选项以只读灰色标签展示
- 输入框 + 添加按钮 + 保存按钮 全部 **disabled**
- 代码保留不动，方便以后恢复

## 立刻推送逻辑变更

现有 `pushFeishuNow()` 直接调 `FeishuSync.pushNow()`，没有和自动推送定时器交互。新逻辑：

```
pushFeishuNow():
  ① stopFeishuTimers()           // 暂停自动推送定时器
  ② 执行推送请求
  ③ .then() / .catch() 中:
     refreshFeishuStatus()       // 刷新状态（会重启定时器）
```

效果：手动推送不会和自动推送冲突，推送完成后倒计时重新开始。

## 关键注意点

### localStorage 兼容

旧用户可能存有 `activeSettingsSub` 值。由于班次设置从索引 5 移到索引 0，旧值会指向错误的子页面。`restoreSettingsSub()` 中已有范围校验 `idx >= 0 && idx <= 7`，只需把上限从 6 改为 7。首次加载时旧值超出范围会 fallback 到 0（新顺序下班次设置），恰好合理。

### 开关替换勾选框

原代码通过 `document.getElementById('s-feishu-sync')` 读写 checkbox 状态。改为 iOS 开关后：
- 开关点击事件调用 `toggleFeishuSync(!currentState)`
- `updateFeishuStatusUI` 中 `toggle.checked = status.enabled` 改为操作开关 CSS class
- 开关 DOM 元素使用独立 ID（如 `feishu-toggle`）

### 子页面 DOM 顺序决定索引

`switchSettingsSub(i)` 使用 `querySelectorAll('.settings-tab')` 返回 NodeList，其索引按 DOM 顺序。因此重排 HTML 后索引自动跟随，不需要改 JS 中的循环逻辑。唯一的硬编码 `if (i === 6) { loadSaveList(); }` 因为数据管理仍在新索引 6 所以无需改动。

### 异常标记功能保留

异常标记选项的 JS 函数（`addExceptionOption`、`saveExceptionOptions`、`loadExceptionOptions`）全部保留，仅 UI 层 disabled。将来需要启用时删除 `disabled` 属性 + 去掉「暂不可用」标签即可。

## 不在范围内

- 飞书同步子页面的**拖拽排序**功能（用户提到可能以后加，本次不做）
- 异常标记选项的**功能恢复**（暂时只读，不做后端改动）
- 自动推送间隔的**实际生效**（只读展示，不接入后端）
- 主标签页顺序调整（只改设置子页面，不动顶部导航）
