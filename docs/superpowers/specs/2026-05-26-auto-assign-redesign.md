# 自动分配功能完全重新设计

## 背景

当前自动分配存在 40+ 问题：参数过多且含 bug 补丁、交互反人类（必须先预览才能确认）、预览/确认参数不一致、无错误处理、N+1 查询性能差、优先级硬编码、全内联样式无 CSS 类。

本次完全重新设计，从根本上简化交互、修复所有已知 bug、内建跨班次能力。

## 设计概览

### 弹窗布局

三组可折叠区域 + 高级选项：

```
┌─────────────────────────────────────────┐
│ 自动分配任务          默认值已就绪      │
├─────────────────────────────────────────┤
│ ▸ 时间范围    从现在开始 · 不限制结束   │
│                                         │
│ ▾ 机器              已选 3 台          │
│   [全部] [M1] [M2] [常规] [特殊]       │
│   ┌─────────────────────────────────┐   │
│   │ ✓ 机器A        M1 · 常规       │   │
│   │ ✓ 机器B        M1 · 常规       │   │
│   │ ✓ 机器C        M2 · 特殊       │   │
│   │ - 机器D        M2 · 维修停用    │   │
│   │ ☐ 机器E        M2 · 常规       │   │
│   └─────────────────────────────────┘   │
│                                         │
│ ▾ 任务       4个待分配 · 按优先级排列   │
│   [全部] [M1] [M2] [常规] [特殊]       │
│   ┌─────────────────────────────────┐   │
│   │ ✓ P0 紧急采集A   M1 · 常规 · 2h │   │
│   │ ✓ P1 日常巡检B   M1 · 常规 · 1h │   │
│   │ ✓ P1 数据采集C   M2 · 特殊 · 3h │   │
│   │ ✓ P2 补充采集D   M2 · 常规 ·30m │   │
│   └─────────────────────────────────┘   │
│                                         │
│ ▸ 高级    间隔0分钟 · 休息段默认策略    │
├─────────────────────────────────────────┤
│            [取消] [预览分配] [确认分配]   │
└─────────────────────────────────────────┘
```

### 时间范围（可折叠，默认收起）

"从"和"到"各自独立，下拉切换：

| 字段 | 默认值 | 可选值 |
|------|--------|--------|
| 从 | 现在 | 现在 / 指定时间（出现 date+time 控件） |
| 到 | 不限制 | 不限制 / 指定时间（出现 date+time 控件） |

- 常见组合：从现在到不限制、从现在到指定结束、指定开始到指定结束（跨夜班）
- "恢复默认"一键清空

### 机器/任务选择（默认展开）

- **Tab 过滤**：全部、按机型（M1/M2/...）、按类型（常规/特殊/...）
- **列表式**：checkbox + 名称 + 元信息，维修停用灰色不可选
- **任务排序**：从 config 表读取 `priorities` 的 `sort_order`，上方优先先分配。新增/删除/重排优先级后自动适配

### 高级选项（可折叠，默认收起）

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 任务间隔 | 数字(分钟) | 0 | 任务之间留空，方便拖拽 |
| 允许任务覆盖休息时段 | toggle | 开启 | 关闭=任务避开休息段 |
| 跨休息段自动延长时长 | toggle | 开启 | 开启=时间段含休息则自动加上 |
| 排除时段 | 动态列表 | 空 | 手动指定不可分配的时间段 |

两个 toggle 正交组合覆盖四组需求：
- 覆盖休息=关：组长A（保证休息）
- 覆盖休息=开：组长B（赶工期）
- 自动延长=开：任务组A（休息算进时长）
- 自动延长=关：任务组B（加急不加时）

## 交互行为

### 预览流程（核心交互）

状态机：

```
空闲 → 打开弹窗 → 配置中
配置中 → 点预览 → 预览中（弹窗关闭，时间轴顶部出现浮动操作栏）
预览中 → 点确认 → 空闲（分配完成，toast 含撤销）
预览中 → 点取消 → 空闲（清除半透明预览卡片）
预览中 → 点返回调整 → 配置中（弹窗恢复之前参数，预览卡片保持）
配置中 → 改参数 → 预览自动清除，卡片消失
配置中 → 点取消 → 空闲（关闭弹窗，如有预览卡片也清除）
空闲（刚分配完）→ 点撤销 → 空闲（排班恢复）
```

**浮动操作栏**（预览中时显示在时间轴顶部）：

- "预览中 | N 个任务待确认"
- "返回调整" — 重开弹窗，恢复参数
- "确认分配" — 直接提交
- "取消预览" — 清除预览卡片，回到空闲状态

**关键规则**：
- 预览时弹窗消失，让用户自由浏览时间轴
- 半透明虚线边框卡片表示待确认的分配
- "返回调整"后修改任何参数 → 半透明卡片自动清除
- 弹窗内点"取消" → 同时清除时间轴上的预览卡片

### 确认

1. 可通过浮动操作栏或弹窗内的"确认分配"按钮触发
2. 使用预览时的参数快照（不再重新读取，修复一致性 bug）
3. 成功后 toast 显示结果 + "撤销"按钮
4. 半透明卡片变实心，浮动栏消失

### 撤销

1. 分配成功后 toast 出现"撤销"按钮
2. 后端返回本次分配的 schedule ID 列表，撤销时按 ID 删除，恢复任务状态为"待分配"
3. 撤销有效期：直到用户做其他操作（新分配/手动编辑/批量延迟等）

### 键盘

- Esc：关闭弹窗
- Enter：触发预览（在弹窗内时）
- 点击遮罩：关闭弹窗

### 错误处理

- fetch 添加 .catch()：网络错误显示 toast "网络异常，请重试"
- 加载中：预览/确认按钮显示 spinner
- 空状态：无机器显示"暂无可选机器"，无任务显示"没有待分配任务"，带图标
- 后端异常：返回具体错误 msg，前端 toast 显示

### 记忆

- 高级选项的设定跨会话记住（存到 config 表或 localStorage）
- 机器/任务选择不记住（每次打开重新加载）
