# 任务动态卡片分组聚合设计

> 优化飞书"任务动态"卡片：从逐条平铺改为按组→机器→包自动分组聚合，多档展示。

## 背景

当前 `build_announcement_card` 将每条事件逐行平铺，仅展示前 10 条，其余折叠为"…等共 N 条"。115 条任务完成时大量信息丢失，且无视觉层次。

## 目标

自动按组→机器→包分组聚合，根据总量自动切换展示粒度：
- 少量（≤10）：保持原样
- 中量（10~50）：分组展开
- 大量（>50）：全局收拢

## 聚合规则

### 底层聚合（包级别，始终生效）

同机器+同包内：
- 完成数 ≥ 2 → `{首个任务名}等{N}个任务已完成`
- 完成数 = 1 → `{首个任务名}已完成`

### 排序键

`(group_name or "未分组", machine_name, package_name or "")`

先按组排 → 组内按机器排 → 机器内按包排。

## 四档展示

### 档位 1：原样（total ≤ 10）

逐条平铺，和当前格式完全一致：

```
1组 BR2-10 6-任务包 06-12 笔袋贴靠笔筒 已完成
1组 BR2-10 6-任务包 06-12 笔筒移到桌面中央 已完成
📊 共 7 个
```

### 档位 2：全量展开（10 < total ≤ 50 且每组 ≤ 10）

组→机器→包→任务，缩进层次：

```
▸ 1组
  BR2-10 · 6-任务包
    笔袋贴靠笔筒等5个任务已完成
  BR2-10 · 9-任务包
    拔下鼻喷剂盖已完成
▸ 2组
  D2-01 · 4-任务包
    检查记录本等3个任务已完成
📊 共 2 组 28 个
```

### 档位 3：单组收拢（10 < total ≤ 50 但某组 > 10）

超限组收拢到机器摘要行，其余组正常展开：

```
▸ 1组 22 个
  BR2-10 15 个任务已完成
  BR2-11 7 个任务已完成
▸ 2组
  D2-01 · 4-任务包
    检查记录本等3个任务已完成
📊 共 2 组 48 个
```

超限组的机器行不显示包级细节。

### 档位 4：全局收拢（total > 50）

所有组收拢到组→机器两层：

```
▸ 1组 42 个
  BR2-10 23 个任务已完成
  BR2-11 19 个任务已完成
▸ 2组 28 个
  D2-01 16 个任务已完成
  D2-03 12 个任务已完成
📊 共 5 组 115 个
```

## 决策树

```
events
  │
  ├─ total ≤ 10 ──→ 档位1：原样逐条
  │
  ├─ 10 < total ≤ 50 ──→ 按组遍历：
  │        │
  │        ├─ 该组 > 10个 ──→ 档位3：组内收拢（只展示机器摘要）
  │        │
  │        └─ 该组 ≤ 10个 ──→ 档位2：全量展开（机器→包→任务）
  │
  └─ total > 50 ──→ 档位4：全局收拢（组→机器）
```

## 边界情况

| 情况 | 处理 |
|------|------|
| 无包名 | 跳过包层级，机器直接到任务 |
| 有包+无包混合 | 无包的归入"独立任务"分组 |
| 无组 | 组名显示"未分组" |
| 无组+无包 | 机器→任务，组名"未分组" |
| 混合事件类型 | 按事件类型用不同颜色标注（已完成/已开始/全部完成） |

## 脚注

所有档位末尾统一加 `📊 共 X 组 Y 个`（≤10 条时只显示 `📊 共 Y 个`）。

## 涉及文件

- `feishu/events/cards.py` — `build_announcement_card()` 重写

## 不变

- 卡片头部 `✅ 任务动态` 不变
- 模板颜色 `CARD_COLORS["announcement"]`（green）不变
- 事件类型颜色映射不变（已完成 `#4CAF50`、已开始 `#2196F3`、全部完成 `#FF9800`）
- 去重逻辑（`push_log`）不受影响，只改展示层
