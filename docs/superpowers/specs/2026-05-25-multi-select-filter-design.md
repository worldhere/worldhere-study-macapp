# 多选筛选设计 Spec

**日期:** 2026-05-25  
**状态:** 已确认

---

## 目标

将机器管理页面的筛选栏从单选 `<select>` 改为多选标签面板模式。每个筛选维度支持同时选择多个值，维度间 AND 关系，不一致条件的机器自动进隐藏区。

## 核心交互

### 筛选栏布局

4 个筛选维度保持一行：**筛选机型 / 状态 / 任务类型 / 分组**。右侧始终显示 `↺ 重置` 按钮，无筛选条件时灰色禁用。

### 点击下拉 → 展开标签面板

点击某个维度的下拉框，在其下方展开该维度的标签选择面板。面板内将所有可选值渲染为可点击标签：

- 已选中的标签：实心填充色（维度专属颜色）
- 未选中的标签：白底灰字空心
- 点击标签即时切换选中/取消，立刻更新机器列表和隐藏区
- 点击面板外部或再次点击下拉框收起面板
- 可以同时打开多个维度的面板

### 条件标签条

面板收起后，当前有效的筛选条件以标签条形式显示在筛选栏下方：

- 每个条件一个彩色标签：`维度: 值 ✕`
- 点 ✕ 删除该条件，等同于取消勾选对应标签
- 不同维度用不同颜色区分（如机型=橙黄，分组=蓝，状态=绿，任务类型=紫）
- 无任何条件时不显示标签条

### 重置按钮

- 筛选栏右侧 `↺ 重置` 始终可见
- 有筛选条件时可点击，一键清除所有条件
- 无筛选条件时灰色禁用
- 重置后所有维度回到"不限"状态，机器列表全量显示
- 重置不影响手动眼睛隐藏的机器

### 空状态规则

- 某个维度全不选 = "不限" = 不过滤该维度
- 所有维度全不选 = 显示全部机器

### 维度间关系

全部 AND：`机型 IN [...] AND 状态 IN [...] AND 任务类型 IN [...] AND 分组 IN [...]`

## 隐藏区联动

- 不符合任何筛选条件的机器自动进入隐藏区（已存在的 `#hidden-machines-module`）
- 隐藏区内每台机器可点击 👁 恢复显示（独立于筛选条件）
- 手动恢复的机器不受后续筛选变更影响 — 被手动恢复后保持可见，直到再次点击眼睛隐藏
- 重置不影响手动眼睛隐藏/恢复的状态

## URL 同步

- 多选值通过 URL 查询参数同步：`?m_type=BR1&m_type=Mini&m_group=A组&m_group=B组`
- 使用 `request.args.getlist()` 读取多值
- 支持复制链接分享当前筛选视图

## 涉及文件

| 文件 | 改动 |
|------|------|
| `templates/panels/machines.html` | 筛选栏 DOM 改造：下拉框 + 标签面板容器 + 条件标签条 + 重置按钮 |
| `static/style.css` | 标签面板样式、条件标签条样式、选中/未选中态、维度颜色 |
| `static/core.js` | `_filterMachinesByUI()` 改为多值筛选 + 隐藏区联动 |
| `static/machines.js` | 标签面板展开/收起/选中逻辑、条件标签条渲染与操作、重置逻辑 |
| `routes/views.py` | `request.args.get()` → `getlist()` |
| `models.py` | 如后端筛选启用，改为 `IN` 查询（当前死代码，不强制改） |

## 非目标

- 不改变后端 API `/api/machines` — 仍返回全量数据，筛选全在客户端
- 不改变眼睛图标的隐藏机制 — 只是多了筛选自动隐藏这条路径
- 不在本次实现排序/搜索/pagination
