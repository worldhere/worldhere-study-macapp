# 指派任务弹窗重设计

## 背景

当前指派弹窗是最老版的纯表单：固定定位 modal，只有日期、机器下拉、开始/结束时间五个字段。没有机器排班预览、没有空闲时段可视化、没有智能推荐。用户指派时看不到目标机器已有的任务分布，全靠猜。

## 目标

重新设计指派弹窗，核心原则：**少操作、智能推荐、高优先级（覆盖冲突）、好看**。

## UI 设计

### 弹窗结构（从上到下）

1. **标题栏** — 对齐现有 `drawer-header` 风格：浅色背景、蓝色图标块、标题 + 关闭按钮
2. **任务信息条** — 任务名、类型/种类/优先级、预估时长 badge
3. **表单区** — 两行：
   - 第一行：目标机器下拉（全宽），推荐项标 ★推荐
   - 第二行：开始时间 + 结束时间并排，各含 `date` + `time` 原生输入
4. **🔄 切换推荐组合** 按钮 — 虚线边框，低视觉权重
5. **Mini 时间轴** — 纯展示，镜像主视图
6. **冲突提醒** — 黄色警告条
7. **操作按钮** — 取消 + 确认指派·覆盖冲突（蓝色渐变 + 阴影）

### 弹窗宽度

~680px，比原 400px 放宽以容纳时间轴和双列字段。

### 风格要点

- 标题栏对齐现有 `drawer-header`（非暗色）
- 推荐字段绿色边框高亮
- 结束时间灰色文字表示"自动计算"
- 冲突提醒黄色背景

## 时间输入

使用原生 HTML 控件：

| 字段 | 控件 | 说明 |
|---|---|---|
| 开始日期 | `<input type="date">` | 默认今天 |
| 开始时间 | `<input type="time">` | 推荐值绿色高亮，自动填最早空档 |
| 结束日期 | `<input type="date">` | 自动 = 开始日期 + 跨天偏移 |
| 结束时间 | `<input type="time">` | 自动 = 开始时间 + 预估时长 |

开始日期/时间变更时自动重算结束日期/时间。时间选择器为浏览器原生控件（Chrome/Edge 自带时钟面板），无需额外代码。

## Mini 时间轴

### 核心原则：纯展示，无交互

- 不响应点击
- 绿色虚线框随表单输入实时更新位置
- 完全复用主时间轴的渲染基础设施

### 复用策略

| 元素 | 来源 |
|---|---|
| 坐标系 | `abs_min` + `minToPx()` |
| 视图范围 | `getViewRange()` — 夜班/双班/自定义自动同步 |
| 小时宽度 | `--hourWidth` — 主时间轴缩放时同步 |
| 日期/小时头 | `_hourLabelsForRange()` |
| 任务色块 | `_createTaskBlock()` 去交互化（无 resize/按钮/拖拽） |
| now 红线 | `renderCurrentTimeMarker` 逻辑简化版 |

### 布局

- 外层 `overflow-x: auto`，可水平滚动
- 内容宽度 = `24 * hourWidth`（跟随缩放）
- 单轨道高度 ~52px
- 已有任务在轨道上半行，绿色虚线框跨全轨道高度

### 任务块裁剪

从 `_createTaskBlock()` 生成的块中移除：
- `resize-left` / `resize-right` 手柄
- `complete-btn` / `recall-btn` 按钮
- `mousedown` / `dragstart` / `dragend` / `dblclick` 事件

保留：
- `task-type-N` class（颜色）
- 任务名 + 时间文字
- `task-completed` / `task-split` / `task-incompatible` 状态样式

## 推荐算法

### 输入

- 任务类型（`task.type`）
- 任务种类（`task.task_kind`）
- 预估时长（`task.est_seconds` → 分钟）
- 候选机器：类型匹配 AND 非维修停用

### 空闲槽位扫描

对每台候选机器：
1. 取该机器当天的 schedule，按时段排序
2. 从 `max(当前时间, 00:00)` 开始扫描
3. 相邻任务之间的 gap ≥ 预估时长 → 可用槽位
4. 最后一个任务结束后到 24:00 也视为可用
5. 当天无排班 → 整天空闲

### 排序策略：负载均衡 + 时间最早

```
机器当日任务数 ASC → 槽位开始时间 ASC
```

### 🔄 切换行为

| 操作 | 行为 |
|---|---|
| 切换推荐组合 | 跳到列表中下一组（机器, 槽位），更新下拉 + 时间字段 + 绿色框 |
| 到头 | 循环回第一个 |
| 手动改机器 | 重置为该机器的第一个推荐槽位 |
| 手动改时间 | 退出推荐模式，"★推荐"标记消失 |

## 冲突处理

复用现有后端逻辑：`/assign_task` 已实现"自动消除重叠并顺延后续任务"。前端展示黄色警告条告知用户哪些任务会被顶掉。
