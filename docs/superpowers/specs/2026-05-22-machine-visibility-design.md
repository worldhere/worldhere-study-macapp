# 机器可见性交互重构

## Context

机器管理表格中的"可见"列用于控制在时间轴面板上显示/隐藏机器。当前用 checkbox 实现，交互体验差：
- checkbox 在表格中视觉丑陋，像数据选择工具
- 隐藏后的机器仍在主表格中显示（仅取消勾选），无专门区域管理

用户期望：眼睛图标 + 即点即隐藏 + 被隐藏的机器在独立表格模块中展示。

## Design

### 两个独立表格模块

**主表格** — "机器列表"：显示匹配筛选条件且未被手动隐藏的机器。时间轴上可见。

**已隐藏机器表格** — "已隐藏的机器"：显示匹配筛选条件但被手动隐藏的机器。不在时间轴上显示，但其任务仍在进行中。这是一个完整功能的表格（与主表格相同的列和操作按钮），用户可以对隐藏的机器进行状态切换、任务类型修改、删除等操作。

隐藏表格可折叠，点击模块标题栏展开/收起。

### 筛选与隐藏的关系

- **筛选优先**：两个表格都只展示匹配当前筛选条件（机型/状态/任务类型）的机器
- **手动隐藏叠加**：在筛选结果中，被手动隐藏的机器从主表格移到隐藏表格
- 切换筛选条件时，两个表格同步更新

### 眼睛图标

- 👁 睁眼：机器在时间轴可见，点击后进入隐藏表格
- 👁‍🗨 闭眼：机器已隐藏，点击后恢复到主表格
- 替换原来"可见"列的 checkbox

### 全部恢复

隐藏表格底部有"全部恢复显示"按钮，一键将当前筛选下所有隐藏机器移回主表格。

### 空状态

- 无隐藏机器时：隐藏表格模块整个不渲染
- 主表格为空（所有机器被隐藏）：显示空状态提示

## Affected Files

- `static/timeline.js` — `_renderMachineTable` 拆分为主表格渲染和隐藏表格渲染；checkbox 改为眼睛图标
- `static/machines.js` — `_appendMachineRow` 中 checkbox 改为眼睛图标
- `templates/panels/machines.html` — 添加隐藏表格容器 DOM
- `static/core.js` — `_toggleMachineVisibility` 逻辑基本不变

## Edge Cases

- 新增机器默认可见，不进入隐藏表格
- 删除隐藏表格中的机器：正常删除，同时清理 `_hiddenMachineIds`
- 筛选条件切换后，隐藏表格中只显示"匹配筛选 + 已隐藏"的机器
- 隐藏表格折叠状态不需要持久化（每次刷新默认展开）
