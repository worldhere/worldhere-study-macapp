# 飞书应用权限配置指南

> 在飞书开放平台 <https://open.feishu.cn> 创建新应用后，按本指南配置权限。

## 一、应用身份权限（tenant_access_token）

应用以自己身份调用 API 时需要的权限。以下所有权限都必须开通，否则对应功能直接不可用。

### 核心（5 个，缺一不可）

| 权限标识 | 权限名称 | 调用的 API | 对应功能 |
|----------|----------|-----------|----------|
| `bitable:app` | 多维表格 | `/bitable/v1/apps/{app_token}/tables/*` | 排班数据读写、建表、字段管理、初始化、同步 |
| `im:message` | 发送消息 | `/im/v1/messages` | 推送任务提醒、班次报告、变动通知、私信 |
| `im:image` | 上传图片 | `/im/v1/images` | 上传排班截图到飞书（时间轴截图 & 表格截图） |
| `im:chat` | 获取群信息 | `/im/v1/chats` | 读取已配置的推送群列表 |
| `contact:user` | 获取用户信息 | `/contact/v3/users` | 读取用户姓名和 open_id，组长/采集员通知路由 |

### 推荐（2 个，外部用户场景必需）

| 权限标识 | 权限名称 | 调用的 API | 对应功能 |
|----------|----------|-----------|----------|
| `contact:user.id:readonly` | 通过手机号或邮箱获取用户 ID | `/contact/v3/users/batch_get_id` | 外部人员提供手机号即可查出 open_id，无需手动复制 |
| `im:chat:write` | 创建群聊 | `/im/v1/chats` (POST) | 机器人自动建群（已实测可用） |

### 可选（1 个）

| 权限标识 | 权限名称 | 调用的 API | 对应功能 |
|----------|----------|-----------|----------|
| `im:chat:member` | 管理群成员 | `/im/v1/chats/{chat_id}/members` | 拉人入群、移出群聊 |

### 不需要开通

以下权限代码库从未调用，无需勾选：

`calendar` `drive:file` `wiki` `docx:document` `contact:department` `mail` `task` `approval` 等

## 二、用户身份权限（user_access_token）

以用户身份调用 API 时需要的权限。当前项目**所有 API 调用均使用 tenant_access_token**，无需开通用户身份权限。

如果将来需要以下功能，需额外配置：

| 场景 | 需要的用户权限 | 说明 |
|------|-------------|------|
| 以用户身份发送消息 | `im:message` | 消息发送者显示为用户而非机器人 |
| 读取用户云文档 | `drive:file` | 访问"我的空间"的文档 |
| 读用户日历 | `calendar:calendar` | 读取个人日程 |

> **注意**：用户身份权限需要用户 OAuth 授权才能生效，不能用 tenant_access_token 代替。

## 三、创建新应用步骤

1. 打开 [飞书开放平台](https://open.feishu.cn)
2. 创建企业自建应用 → 填写应用名称和描述
3. **添加应用能力**：无需勾选（不是机器人类型的"应用能力"，那个可以不开）
4. **权限管理** → 按上表勾选所有核心 + 推荐权限
5. **安全设置** → 添加 IP 白名单（可选）
6. **版本管理** → 创建新版本 → **发布**（不发布权限不生效）
7. 发布后获取 `App ID`（凭证与基础信息页）和 `App Secret`
8. 在设置面板填入凭证，或代码中替换 `feishu_token.py` 的默认值

---

## 四、获取多维表格 Token

1. 在飞书客户端创建多维表格（或在"云文档"中新建 Base）
2. 打开表格 → 浏览器地址栏复制 `/base/` 后到 `?` 前的字符串
3. 这就是 `App Token`，填入设置面板的「App Token」字段
