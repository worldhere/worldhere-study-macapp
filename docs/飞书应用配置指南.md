# 飞书应用配置指南

从头创建一个飞书应用（含机器人），拿到凭证和群聊 ID 填入排班系统。

---

## 第一步：进入飞书开放平台

浏览器打开：**https://open.feishu.cn**

用飞书账号登录。
---

## 第二步：创建应用

1. 登录后进入「开发者后台」
2. 点击右上角 **「创建应用」**
3. 应用名称填 `喜欢的名字‘
4. 应用图标随便选一个
5. 点击 **「创建」**

页面会跳转到应用详情页。

---

## 第三步：添加机器人

应用需要一个机器人才能在群里发消息。

左侧菜单 → **「应用功能」** → 找到 **「机器人」** → 点击 **「启用」**

启用后会弹出配置页面：

1. **机器人名称**：填 `排班助手`（或你喜欢的名字）
2. **机器人描述**：填 `排班通知推送`（随意）
3. 点击 **「保存」**

---

## 第四步：拿到 App ID 和 App Secret

左侧菜单 → **「凭证与基础信息」**

页面上能看到：

| 字段 | 说明 | 示例 |
|------|------|------|
| **App ID** | 应用唯一标识 | `cli_aa8ffc77eff89bdb` |
| **App Secret** | 应用密钥，点击「显示」后复制 | `7yncJ4xy9XPXsT6g...` |

把这两个记下来，后面要填入排班系统。

---

## 第五步：开通权限

左侧菜单 → **「权限管理」** → **「API 权限」** 标签页

搜索并开通以下权限：

| 权限 | 用途 |
|------|------|
| `bitable:app` | 操作多维表格（建表、删表） |
| `bitable:app:table` | 读写数据表字段 |
| `bitable:app:record` | 读写数据记录 |
| `im:message` | 机器人发群消息 |
| `im:resource` | 上传图片 |
| `contact:user.id:readonly` | 读取用户信息 |

开通后，点击页面顶部的 **「发布新版本」** → 填写版本号 → **「发布」**。不发布权限不生效。

---

## 第六步：把机器人拉进群，拿到群聊 ID

1. 打开飞书，进入你要接收推送的群聊
2. 群聊右上角 **「设置」** → **「群机器人」** → **「添加机器人」**
3. 搜索你的应用名（如 `排班同步`），添加进去
4. 回到这个群的聊天页面，看浏览器地址栏：

```
https://xxx.feishu.cn/chat/oc_xxxxxxxxxxxxxx
                          ═══════════════════
                            这就是群聊 ID
```

`oc_` 开头的那串就是群聊 ID。复制下来。

> 如果需要推送到多个群，每个群都拉一次机器人，分别复制群聊 ID。

---

## 第七步：创建多维表格并拿到 App Token

1. 飞书左侧菜单 → **「云文档」** → 右上角 **「新建」** → **「多维表格」**
2. 表格名称随便填，比如 `排班数据`
3. 创建好后，看浏览器地址栏：

```
https://xxx.feishu.cn/base/I7IzbOlscajHJZscWOtcYcs6nLf?table=tblXXX
                         ═══════════════════════════
                              这就是 App Token
```

`/base/` 后面那串一直到 `?`（或网址末尾）的字符串就是 App Token。复制下来。

---

## 第八步：填入排班系统

打开排班系统 → 设置 → **「飞书同步」**

### 8.1 填入应用凭证（页面最下面）

| 输入框 | 填什么 | 哪来的 |
|--------|--------|--------|
| App ID | `cli_ 开头那串` | 第四步 |
| App Secret | `应用密钥` | 第四步，点「显示」复制 |
| App Token | `/base/ 后面那串` | 第七步，地址栏复制 |

点击 **「💾 保存」**。

### 8.2 配置推送群聊（推送设置区域）

1. 打开 **「推送设置」** 开关
2. 在 **群聊列表** 里填入群名称和群聊 ID：

| 输入框 | 填什么 | 示例 |
|--------|--------|------|
| 群名称 | 自己起的名字，方便辨认 | `白班组` |
| oc_xxxx | 群聊 ID | `oc_xxxxxxxxxxxxxx` |

3. 点击 **「➕ 添加」**，可以添加多个群
4. 在 **「推送事件设置」** 表格里勾选每个群要接收哪些通知
5. 点击 **「🚀 发送」** 测试消息确认机器人能说话

---

## 第九步：开启同步

回到飞书同步页面顶部，打开 **「飞书数据同步」** 开关。

系统会自动：
1. 在飞书多维表格里为每台机器创建一张表
2. 把当前排班数据推送到表格中
3. 之后每 30 秒自动同步
4. 排班变动、异常、班次报告等会自动推送到配置的群里

---

## 常见问题

**Q: 点「保存并验证」提示凭证无效？**

确认 App ID 和 App Secret 没有复制错，确认应用已经发布（第五步那个「发布新版本」按钮）。

**Q: 同步开关点不开？**

先确认应用凭证已保存，再看页面上的状态提示。

**Q: 测试消息发不出去？**

- 确认机器人已经拉到群里（第六步）
- 确认 `im:message` 权限已开通并发布
- 确认群聊 ID 是 `oc_` 开头

**Q: 换了多维表格怎么办？**

把新的 App Token 填进去保存，然后在飞书同步面板点 **「初始化」** 重新建表推数据。

**Q: 机器人发消息提示无权限？**

检查权限管理页面是否已经点「发布新版本」。每次改权限都要重新发布。
